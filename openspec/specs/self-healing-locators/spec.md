## 新增需求

### 需求:生成脚本中注入自愈定位函数

当生成的脚本包含交互步骤时，系统必须在脚本中注入 `_safe(page, strategies)` 辅助函数定义。该函数包含语法策略遍历、指纹记录、指纹恢复三层逻辑。

#### 场景:有多策略交互步骤时注入函数

- **当** 生成的用例包含至少一个交互步骤且 `_build_fallbacks()` 返回策略数 > 1
- **那么** 脚本必须在 `import` 块之后、`def test_` 之前包含 `_safe()` 函数的完整定义
- **那么** `_safe()` 函数定义包含指纹文件路径常量、指纹加载/保存、属性提取/重建、过期清理等子函数

#### 场景:纯导航用例不注入函数

- **当** 生成的用例不包含任何交互步骤（仅有 goto / expect / wait / screenshot 等）
- **那么** 脚本不得包含 `_safe()` 函数定义

### 需求:多策略级联定位

`_safe(page, strategies)` 函数必须按顺序尝试定位策略，并集成指纹层。

#### 场景:首个策略命中并记录指纹

- **当** `_safe()` 被调用且 strategies 列表的第一个选择器在页面上存在至少一个匹配元素
- **那么** 返回该 Locator 对象，不尝试后续策略
- **那么** 成功后提取元素属性并更新指纹记录

#### 场景:首个策略失败，第二个命中

- **当** strategies 列表中第一个选择器在页面上匹配数为 0
- **那么** 自动尝试第二个选择器，如果命中则返回该 Locator，不尝试后续策略
- **那么** 成功后提取元素属性并更新指纹记录

#### 场景:语法策略全部失败，指纹恢复命中

- **当** strategies 列表中所有语法策略匹配数均为 0，但指纹文件中存在对应记录
- **那么** 系统从指纹属性重建 Locator
- **那么** 成功时返回该 Locator，回写新指纹，输出自愈日志

#### 场景:所有策略失败（含指纹）

- **当** strategies 所有策略失败且指纹也无法恢复
- **那么** 返回最后一个策略的 Locator，将失败交给 Playwright 原生报错

### 需求:自愈开关

当某个步骤的备选策略数量为 1（仅原始选择器）时，生成的代码必须退化回原始 `locator().action()` 形式。

#### 场景:单策略步骤退化

- **当** 某步骤的备选推导结果只有 1 个有效策略
- **那么** 生成的代码为 `page.locator(selector).click()` 而非 `_safe(page, strategies).click()`

#### 场景:多策略步骤启用自愈

- **当** 某步骤有 2 个或以上备选策略
- **那么** 生成的代码为 `_safe(page, [...strategies]).action()`
