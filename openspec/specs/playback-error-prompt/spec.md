## 需求

### 需求:回放失败后显示中文错误提示
系统必须在回放失败时，在回放结果页中显示结构化中文错误提示，代替原始的堆栈信息展示，帮助用户快速定位失败原因。

#### 场景:定位器超时
- **当** 回放因 locator 超时（TimeoutError）失败
- **那么** 系统显示中文提示"⏱️ 定位元素超时，等待 {time}ms 后仍未找到"
- **那么** 系统显示失败定位器和对应步骤序号
- **那么** 步骤序号显示在错误卡片头部，形式为标签"第 N 步"

#### 场景:定位器匹配多个元素
- **当** 回放因 strict mode violation 失败（定位器匹配到 2+ 元素）
- **那么** 系统显示中文提示"⚠️ 定位器匹配到多个元素，无法确定操作目标"
- **那么** 系统显示定位器表达式和匹配到的元素列表

#### 场景:页面加载状态超时
- **当** 回放因 wait_for_load_state 超时失败
- **那么** 系统显示中文提示"🔄 页面加载状态超时（{state}），页面请求未完成"
- **那么** 系统显示超时的 state 类型和超时时间

#### 场景:元素断言失败
- **当** 回放因 expect 断言失败
- **那么** 系统显示中文提示"❌ 元素状态断言失败"
- **那么** 系统显示定位器和期望状态

#### 场景:未知错误类型
- **当** 回放失败但错误类型无法识别
- **那么** 系统显示中文提示"未知回放错误"
- **那么** 系统显示原始错误信息文本

#### 场景:保留原始日志
- **当** 回放失败且已显示结构化错误提示
- **那么** 原始 stdout/stderr 日志以可折叠方式展示（默认收起）
- **那么** 用户仍可展开查看完整日志
