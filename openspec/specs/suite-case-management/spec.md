## 需求

### 需求:套件下用例管理
测试用例页面必须是测试套件的二级页面，路由层级为 `/suites/:id`。

#### 场景:通过套件进入用例
- **当** 用户在套件列表点击套件的"详情"链接
- **那么** 系统跳转到 `/suites/:id` 页面
- **那么** 页面显示该套件下的所有用例列表，包含名称、标签、步骤数、创建时间

#### 场景:独立用例入口移除
- **当** 系统加载页面时
- **那么** 侧边栏不存在独立的"测试用例"菜单项
- **那么** `/cases` 路由不再注册，访问 `/cases` 返回 404

#### 场景:用例列表操作
- **当** 用户浏览套件下的用例列表
- **那么** 每条用例显示名称、标签、步骤数、创建时间
- **那么** 每条用例提供"编辑"（跳转到 `/cases/:id`）和"删除"操作

#### 场景:手动新建用例
- **当** 用户点击"手动新建"按钮
- **那么** 系统跳转到 `/cases/new?suite_id=X` 页面，用例编辑器预填 suite_id

### 需求:用例详情编辑与保存
用例详情页必须支持步骤编辑、保存和导航。

#### 场景:编辑步骤
- **当** 用户进入 `/cases/:id` 页面
- **那么** 页面加载用例的步骤列表并通过 StepEditor 以表格形式展示
- **那么** 用户可添加、修改、删除、复制、排序步骤
- **那么** 用户可为每个步骤添加备注文本

#### 场景:复制步骤
- **当** 用户点击操作列的复制按钮
- **那么** 系统深拷贝当前步骤，插入到被复制步骤下方
- **那么** 所有步骤自动重新编号

#### 场景:添加备注
- **当** 用户在备注列的 vxe-input 中输入文本
- **那么** 该步骤的 note 属性实时更新为输入内容
- **那么** 未输入时 note 保持 undefined

#### 场景:保存用例
- **当** 用户点击"保存"按钮
- **那么** 系统通过 `PUT /api/cases/:id` 更新用例的步骤数据（含 note）
- **那么** 保存成功后显示成功提示
