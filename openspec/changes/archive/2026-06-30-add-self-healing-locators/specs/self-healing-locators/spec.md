## 新增需求

### 需求:生成脚本中注入自愈定位函数

当生成的脚本包含交互步骤时，系统必须在脚本中注入 `_safe(page, strategies)` 辅助函数定义。

#### 场景:有多策略交互步骤时注入函数

- **当** 生成的用例包含至少一个交互步骤（click/fill/check/uncheck/select/hover）
- **那么** 脚本必须在 `import` 块之后、`def test_` 之前包含 `_safe()` 函数的完整定义

#### 场景:纯导航用例不注入函数

- **当** 生成的用例不包含任何交互步骤（仅有 goto / expect / wait / screenshot 等）
- **那么** 脚本不得包含 `_safe()` 函数定义

### 需求:多策略级联定位

`_safe(page, strategies)` 函数必须按优先级逐个尝试定位器，命中即返回。

#### 场景:首个策略命中

- **当** `_safe()` 被调用且 strategies 列表的第一个选择器在页面上存在至少一个匹配元素
- **那么** 返回该 Locator 对象，不尝试后续策略

#### 场景:首个策略失败，第二个命中

- **当** strategies 列表中第一个选择器在页面上匹配数为 0
- **那么** 自动尝试第二个选择器，如果命中则返回该 Locator，不尝试后续策略

#### 场景:所有策略失败

- **当** strategies 列表中所有选择器在页面上匹配数均为 0
- **那么** 返回最后一个策略的 Locator，将最终失败交给 Playwright 原生报错

### 需求:自愈开关

当某个步骤的备选策略数量为 1（仅原始选择器）时，生成的代码必须退化回原始 `locator().action()` 形式。

#### 场景:单策略步骤退化

- **当** 某步骤的备选推导结果只有 1 个有效策略
- **那么** 生成的代码为 `page.locator(selector).click()` 而非 `_safe(page, strategies).click()`

#### 场景:多策略步骤启用自愈

- **当** 某步骤有 2 个或以上备选策略
- **那么** 生成的代码为 `_safe(page, [...strategies]).action()`
