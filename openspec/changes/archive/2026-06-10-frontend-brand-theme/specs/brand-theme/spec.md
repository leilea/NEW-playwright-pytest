## 新增需求

### 需求:全局品牌主题变量
系统必须提供一组 CSS 自定义属性，覆盖 Element Plus 默认主题变量，注入品牌主色、字体和视觉参数。主题文件必须位于 `frontend/src/styles/theme.css`。

#### 场景:主色应用
- **当** 任何页面使用 Element Plus 的 `type="primary"` 按钮或链接
- **那么** 其颜色必须基于品牌主色 #4A90D9 及其衍生色阶

#### 场景:字体优先级
- **当** 页面渲染任何文本内容
- **那么** 系统优先使用 Inter 字体，并通过系统字体栈作为 fallback

#### 场景:圆角一致性
- **当** Element Plus 组件渲染带圆角的边框（按钮、输入框、卡片等）
- **那么** 其基础圆角半径必须为 6px

### 需求:页面字体加载
系统必须在 `index.html` 中通过 Google Fonts 的 `<link>` 标签引入 Inter 字体，并使用 `font-display: swap` 确保文字在字体加载期间立即可见。

#### 场景:字体回退
- **当** 网络不稳定导致 Inter 字体加载缓慢
- **那么** 页面文字必须立即使用系统 fallback 字体显示，加载完成后平滑切换

### 需求:内联硬编码样式清理
所有页面和组件中的内联 `style` 属性若硬编码了颜色（如 `#f5f7fa`、`#1e1e1e`）或尺寸值，必须替换为 Element Plus 语义 CSS 变量或迁移至 scoped style 块。

#### 场景:背景色替换
- **当** 页面使用内联 style 设置 `background:#f5f7fa`
- **那么** 必须替换为 `var(--el-bg-color-page)` 或等效语义变量

#### 场景:登录页背景
- **当** 用户访问登录页
- **那么** 页面背景必须使用品牌导向的视觉呈现（渐变或品牌色浅底色），而非纯灰色

### 需求:导航栏品牌标识
主布局（MainLayout.vue）的侧边栏顶部必须显示品牌标识区域，使用品牌主色作为视觉锚点。

#### 场景:品牌区域可见
- **当** 用户查看侧边栏顶部
- **那么** 必须看到品牌标识文字或图标，背景色为品牌主色或其浅色变体

### 需求:表格与卡片可读性
数据表格行高必须增大，卡片阴影必须柔和化，以改善长时间使用的可读性。

#### 场景:表格行高
- **当** Element Plus 表格渲染数据行
- **那么** 每行高度必须不小于 48px（通过 `--el-table-row-height` 或等效变量）

#### 场景:卡片阴影
- **当** Element Plus 卡片组件渲染
- **那么** 其阴影必须比默认样式更轻更柔和
