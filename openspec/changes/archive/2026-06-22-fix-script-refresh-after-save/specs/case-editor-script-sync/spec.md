## 新增需求

### 需求:保存后同步刷新脚本标签页
系统必须在保存操作步骤后将"生成脚本"标签页内显示的脚本内容同步刷新为与数据库最新步骤数据一致的脚本。

#### 场景:编辑已有用例步骤并保存
- **当** 用户在 `/cases/:id` 页面的"操作步骤"标签页编辑步骤（复制、删除、上移、下移等）
- **并且** 用户点击【保存】按钮
- **并且** `PUT /api/cases/:id` 成功
- **那么** 系统必须调用 `GET /api/cases/:id/script` 重新生成脚本
- **那么** "生成脚本"标签页显示的脚本内容必须与已保存的步骤一致

#### 场景:新建用例保存后无需脚本刷新
- **当** 用户在 `/cases/new` 页面新建用例
- **并且** 用户点击【保存】按钮
- **并且** `POST /api/cases` 成功
- **那么** 系统跳转到套件详情页（无需刷新脚本标签页）
