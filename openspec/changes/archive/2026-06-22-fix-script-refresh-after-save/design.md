## 上下文

当前 `CaseEditor.vue` 使用 `watch(() => form.value.steps, ...)` 监听步骤变更并调用 `loadScript()` 刷新"生成脚本"标签页。但此方法从 API 端读取数据库中的步骤来生成脚本——在用户编辑期间（保存前），数据库尚未更新，因此 watch 触发时脚本内容仍是旧的。

用户点击【保存】后，`PUT /api/cases/{id}` 将新步骤写入数据库，但 `save()` 函数没有调用 `loadScript()`，导致"生成脚本"标签页一直显示保存前的旧脚本。

## 目标 / 非目标

**目标：**
- 保存操作步骤后自动刷新"生成脚本"标签页，使其显示最新保存的步骤对应的脚本

**非目标：**
- 不改变客户端实时生成脚本的方式（仍通过 API 从数据库读取生成）
- 不改变 watch 行为（编辑期间仍从数据库加载，用户未保存时显示旧脚本是预期行为）

## 决策

**决策：在 `save()` 中 PUT 成功后调用 `loadScript(caseId)`**

这是最小侵入性的修复方案。`loadScript()` 函数已经存在，直接复用。在 PUT 成功后、数据库已更新的时间点调用，确保 API 返回的脚本基于最新步骤。

**备选方案（不采用）：**
- 客户端直接生成脚本：需要在 TypeScript 端复制 `script_gen.py` 的逻辑，维护成本高，且脚本始终反映本地编辑而非已持久化状态
- 禁用 watch 中的 loadScript：会改变现有行为，且与本次修复目标无关

## 风险 / 权衡

- [低风险] 多一次网络请求（PUT + GET/script），但脚本生成是轻量操作，无明显性能影响
