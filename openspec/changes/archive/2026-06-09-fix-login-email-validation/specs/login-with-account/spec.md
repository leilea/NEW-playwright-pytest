## 修改需求

### 需求:用户登录账号

用户必须能够使用任意字符串作为登录凭证，系统禁止对登录账号进行 email 格式校验。

#### 场景:使用带句点的邮箱登录
- **当** 用户输入 `admin@local.dev` 作为登录账号
- **那么** 系统必须允许登录，返回 token

#### 场景:使用不带句点的本地账号登录
- **当** 用户输入 `admin@local` 作为登录账号
- **那么** 系统必须允许登录，返回 token

#### 场景:使用纯用户名登录
- **当** 用户输入 `admin` 作为登录账号
- **那么** 系统必须允许登录，返回 token（前提是账号存在）

### 需求:登录表单文案

登录表单的字段标签禁止显示"邮箱"，必须显示"登录账号"。

#### 场景:前端显示
- **当** 用户打开登录页
- **那么** 输入框标签必须显示"登录账号"而非"邮箱"
