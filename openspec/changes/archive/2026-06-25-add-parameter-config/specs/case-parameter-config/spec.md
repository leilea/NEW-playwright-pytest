## 新增需求

### 需求:参数配置区域展示
系统必须在测试用例详情页（`/cases/:id` 和 `/cases/new`）的表单区和标签页之间展示「参数配置」区域，标题旁展示提示语「参数名输入格式示例: username 或 {{username}}」。

#### 场景:进入用例详情页
- **当** 用户访问 `/cases/:id` 页面
- **那么** 页面必须展示「参数配置」区域
- **那么** 标题旁边必须展示提示语「参数名输入格式示例: username 或 {{username}}」

#### 场景:新建用例页面
- **当** 用户访问 `/cases/new` 页面
- **那么** 页面必须展示「参数配置」区域

### 需求:参数增删改
系统必须支持在参数配置区域添加、修改和删除参数行。每行包含三个字段：参数名、参数值、描述。

#### 场景:添加参数
- **当** 用户点击「添加参数」按钮
- **那么** 系统在参数列表末尾追加一行空参数（key、value、description 均为空字符串）

#### 场景:修改参数
- **当** 用户修改参数名输入框的值
- **那么** 该行参数的 key 必须同步更新

#### 场景:删除参数
- **当** 用户点击某行的删除按钮
- **那么** 该行参数必须从列表中移除

#### 场景:空列表提示
- **当** 参数列表为空
- **那么** 系统展示提示文字「暂无参数，点击"添加参数"按钮添加」

### 需求:动态值类型选择
参数值输入框右侧必须提供动态值类型选择按钮（✨），点击弹出下拉菜单展示 10 种动态类型。

#### 场景:选择无偏移量类型
- **当** 用户点击参数值输入框旁边的类型选择按钮
- **并且** 从下拉菜单中选择「随机数字」
- **那么** 系统将 `{{random:6}}` 追加到当前参数值末尾

#### 场景:选择有偏移量类型
- **当** 用户点击参数值输入框旁边的类型选择按钮
- **并且** 从下拉菜单中选择「时间+小时」
- **那么** 系统弹出偏移量设置对话框

### 需求:偏移量设置
对于时间±N小时、日期±N天四种动态类型，系统必须弹出偏移量设置对话框让用户输入偏移值。

#### 场景:设置偏移量并确认
- **当** 在偏移量对话框中输入数字 3
- **并且** 用户点击「确认」按钮
- **那么** 系统将类似 `{{timeAdd:h+3}}` 的占位符追加到当前参数值末尾

### 需求:参数持久化
系统必须将参数数据随用例保存到数据库。

#### 场景:创建用例时保存参数
- **当** 用户在新建页面添加参数后点击「保存」
- **那么** 参数数据必须随 `POST /api/cases` 请求发送并持久化

#### 场景:编辑用例时更新参数
- **当** 用户在编辑页面修改参数后点击「保存」
- **那么** 参数数据必须随 `PUT /api/cases/:id` 请求发送并更新

#### 场景:加载已保存参数
- **当** 用户访问已有用例的详情页
- **那么** 参数配置区域必须展示已保存的参数数据

### 需求:参数占位符替换
系统必须在脚本生成和回放时，将步骤输入值中的占位符替换为实际参数值。

#### 场景:用户自定义参数替换
- **当** 某参数定义为 `{key: "username", value: "admin"}`
- **并且** 步骤输入值中包含 `{{username}}` 或独立的 `username` 单词
- **那么** 生成脚本和回放时 `{{username}}` 和 `username` 必须被替换为 `admin`

#### 场景:内置动态占位符替换
- **当** 步骤输入值中包含 `{{date}}`
- **那么** 生成脚本和回放时 `{{date}}` 必须被替换为当天的实际日期

#### 场景:自定义参数优先于内置占位符
- **当** 用户定义了 key 为 `date` 的参数
- **并且** 步骤输入值中包含 `{{date}}`
- **那么** `{{date}}` 必须被替换为用户自定义的值，而非当前日期

### 需求:参数数据模型
数据库必须支持存储参数配置数据。

#### 场景:存储格式
- **当** 参数数据保存到数据库
- **那么** 必须以 JSON 数组格式存储在 `catalog.cases.parameters` 列中，每项包含 `key`、`value`、`description` 字段
