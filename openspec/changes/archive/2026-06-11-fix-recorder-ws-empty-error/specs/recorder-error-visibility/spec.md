## 新增需求

### 需求:录制错误信息完整暴露
系统必须将录制过程中的完整异常信息（包含 Python traceback）发送到前端，并写入后端日志。

#### 场景:后端异常时前端显示具体错误
- **当** 录制过程中 `start_codegen` 抛出异常
- **那么** 前端必须显示异常的具体消息（而非默认的"录制错误"）

#### 场景:后端异常时日志记录完整 traceback
- **当** `_run_recorder` 捕获到异常
- **那么** 后端日志必须包含完整的 `traceback.format_exc()` 输出

### 需求:重复点击录制按钮被防护
系统必须防止用户重复点击"开始录制"时创建多个 WebSocket 连接。

#### 场景:录制中再次点击开始按钮
- **当** 用户正在录制中并再次点击"开始录制"
- **那么** Playwright 浏览器必须仍然只有一个实例在运行

### 需求:停止按钮在处理已关闭连接时不报错
系统必须优雅处理 WebSocket 在停止时已处于关闭状态的情况。

#### 场景:WS 已关闭时点击停止
- **当** WebSocket 连接已经关闭，用户点击"停止"
- **那么** 系统不得抛出异常或显示错误提示
