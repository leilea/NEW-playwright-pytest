## 修改需求

### 需求:录制过程
用户输入目标 URL 并点击"开始录制"后，系统必须启动 Playwright 浏览器并自动导航到指定 URL。录制完成后生成的 pytest 脚本必须正确解析展平格式的步骤参数。

#### 场景:启动录制
- **当** 用户在 `/suites/:id` 页面点击"录制新用例"按钮
- **那么** 系统展开 RecorderPanel 组件，显示 URL 输入框和"开始录制"按钮

#### 场景:录制过程
- **当** 用户输入目标 URL 并点击"开始录制"
- **那么** 前端通过 WebSocket (`/ws/rec`) 发送 `{cmd: "start", url: "..."}` 到后端
- **那么** 后端启动 playwright codegen 进程
- **那么** 浏览器启动后必须自动导航到用户输入的目标 URL，地址栏必须显示该 URL
- **那么** 录制过程中，前端实时接收 `{event: "step", step: {action, params}}` 并展示步骤列表

#### 场景:URL 为空时录制
- **当** 用户未输入 URL 即点击"开始录制"
- **那么** 录制进程必须立即终止并向前端发送错误消息，禁止启动浏览器

#### 场景:保存录制结果
- **当** 用户点击"生成用例"并输入用例名称
- **那么** 系统通过 `POST /api/cases` 将 {suite_id, name, steps} 保存为新的测试用例
- **那么** 用例列表自动刷新，新用例出现在列表中

#### 场景:录制步骤的脚本生成兼容展平格式
- **当** 步骤以展平格式（如 `{action: "click", selector: "#btn"}`）存储
- **那么** `script_gen.py` 的 `generate_script()` 必须正确读取展平格式步骤中的所有参数
- **那么** 生成的 pytest 脚本中，`page.click("#btn")` 必须包含正确的选择器参数，禁止生成空串 `page.click("")` 或丢失参数
