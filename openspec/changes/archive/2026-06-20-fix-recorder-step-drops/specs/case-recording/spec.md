## 新增需求

### 需求:录制步骤跨页面持久化
系统必须在页面跳转（表单提交、链接点击等）前后均能完整捕获用户操作，禁止因页面导航导致步骤丢失。

#### 场景:表单提交后步骤不丢失
- **当** 用户在录制过程中填写表单并点击提交按钮，页面跳转到新页面
- **那么** 提交按钮的点击步骤必须在页面跳转前被捕获并发射
- **那么** 新页面加载后，用户在新页面上的操作必须继续被正常捕获

#### 场景:点击链接导航后步骤不丢失
- **当** 用户在录制过程中点击链接跳转到新页面
- **那么** 点击链接的步骤必须在页面跳转前被捕获
- **那么** 新页面上的后续操作必须正常录制

### 需求:多字段填写独立捕获
系统必须独立捕获每个输入字段的填充操作，禁止因用户快速切换输入字段而导致前面字段的填写步骤丢失。

#### 场景:连续填写多个字段
- **当** 用户在录制过程中先填写字段 A，然后立即切换到字段 B 填写
- **那么** 录制结果中必须包含字段 A 的 `fill` 步骤和字段 B 的 `fill` 步骤

#### 场景:快速输入不丢失
- **当** 用户在录制过程中快速连续在多个字段中输入文本
- **那么** 每个字段各自的 `fill` 步骤必须都被捕获

### 需求:录制异常可观测
系统必须在后台日志中记录录制过程中的异常信息，禁止静默丢弃错误。

#### 场景:队列刷新异常被记录
- **当** `flush_queue` 中 `page.evaluate` 因任何原因失败
- **那么** 异常信息必须写入 stderr 日志，而非静默丢弃
