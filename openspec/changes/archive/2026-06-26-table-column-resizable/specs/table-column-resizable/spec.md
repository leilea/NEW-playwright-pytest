## 新增需求

### 需求:表格列宽可拖拽调整
系统用例列表（`/suites`）和测试用例列表（`/suites/:id`）中的 el-table 表头列必须支持鼠标拖拽调整列宽。

#### 场景:拖拽表头右边框加宽列
- **当** 用户将鼠标悬停在表头列的右侧边框上
- **那么** 光标变为 `col-resize`（双向箭头）
- **当** 用户按住鼠标左键并向右拖动
- **那么** 该列宽度实时增加，其他列宽度不变，表格总宽度可能超出容器

#### 场景:拖拽表头右边框收窄列
- **当** 用户将鼠标悬停在表头列的右侧边框上并按住鼠标左键向左拖动
- **那么** 该列宽度实时减小，但不低于最小宽度 60px

#### 场景:操作列不可拖拽
- **当** 用户将鼠标悬停在操作列的右侧边框上
- **那么** 光标不变为 `col-resize`，拖拽操作不生效

### 需求:表格列设置默认宽度
系统用例列表和测试用例列表中的所有列必须有固定的初始宽度（使用 `width` 属性替代 `min-width`）。

#### 场景:系统用例列表默认列宽
- **当** 用户打开 `/suites` 页面
- **那么** 各列初始宽度为：序号 60px、系统名称 250px、版本号 120px、描述 300px、系统创建日期 180px、操作 280px

#### 场景:测试用例列表默认列宽
- **当** 用户打开 `/suites/:id` 页面
- **那么** 各列初始宽度为：序号 60px、名称 220px、标签 180px、步骤数 80px、创建时间 180px、操作 160px
