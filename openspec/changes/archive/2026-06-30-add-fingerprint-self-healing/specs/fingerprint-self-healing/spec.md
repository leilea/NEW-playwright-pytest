## 新增需求

### 需求:成功交互后记录元素属性指纹

当 `_safe()` 通过语法策略成功定位到元素、且后续交互操作完成时，系统必须提取该元素的稳定 DOM 属性并存入指纹文件。

指纹 key 为 `md5(route + selector[:80])[:16]`，其中 route 为 `page.url` 去掉 query string 后的路径部分，selector 为 `strategies[0]`（主选择器字符串）。

记录模式为 upsert：同一 key 已存在时更新 `attrs`、`last_seen`，递增 `hit_count`。

#### 场景:首次交互记录指纹

- **当** `_safe()` 首次在某路由上成功定位主选择器为 `__role:button:登录` 的元素
- **那么** 系统提取该元素的 data-testid、aria-label、placeholder、text、role、id、class 属性
- **那么** 用 `md5("/login__role:button:登录")[:-16][:16]` 作为 key 存入 fingerprints.json
- **那么** `hit_count` 初始值为 1

#### 场景:重复交互更新指纹

- **当** `_safe()` 再次在同一路由上成功定位同一选择器的元素
- **那么** 系统重新提取属性并覆盖该 key 的 `attrs` 和 `last_seen`
- **那么** `hit_count` 递增 1

#### 场景:DOM 属性为空时不记录空值

- **当** 被定位元素不存在 data-testid 属性
- **那么** 指纹中 `data-testid` 字段不被写入（而非写入空字符串或 null）

### 需求:语法 fallback 失败后从指纹恢复

当 `_safe()` 的所有语法策略（strategies[0:-1]）在页面上匹配数均为 0 时，系统必须尝试从指纹文件中恢复该选择器的历史属性，并用属性重建 Locator。

#### 场景:指纹命中且属性定位成功

- **当** 语法策略全部失败
- **那么** 且指纹文件中存在 `md5(route + selector[:80])[:16]` 对应记录
- **那么** 且记录的属性中包含 `data-testid` 字段
- **那么** 系统构建 `page.get_by_test_id(value)` 并验证 `count() > 0`
- **那么** 成功时返回该 Locator，并**回写新指纹**（DOM可能已变）
- **那么** 输出版本日志 `[fingerprint] healed: __role:button:登录 via data-testid=submit-login`

#### 场景:指纹命中但属性全部失效

- **当** 语法策略全部失败
- **那么** 且指纹记录存在但其所有属性构建的 Locator `count()` 均为 0
- **那么** 系统降级到 `page.locator(strategies[-1])`，由 Playwright 原生报错

#### 场景:无指纹记录时直接降级

- **当** 语法策略全部失败且指纹文件中无对应 key
- **那么** 系统直接返回 `page.locator(strategies[-1])`，不执行任何 JSON 读取操作

### 需求:指纹文件体积控制

指纹文件必须内置体积控制机制，防止无限膨胀。

#### 场景:条目数超过上限时淘汰最旧条目

- **当** 指纹文件条目数超过 1000 条
- **那么** 系统按 `last_seen` 排序，淘汰最早的那组条目直到 ≤ 1000 条
- **那么** 淘汰发生在保存文件之前

#### 场景:超过 90 天未命中的条目自动清理

- **当** 某条目的 `last_seen` 距当前时间超过 90 天
- **那么** 系统在加载文件时从内存中移除该条目
- **那么** 该条目不会在下次保存时写回

#### 场景:JSON 文件损坏时降级为空库

- **当** fingerprints.json 内容无法解析为合法 JSON
- **那么** 系统打印警告日志，以空指纹库继续运行
- **那么** 本次运行仍正常记录新指纹，保存时覆盖损坏文件

### 需求:指纹功能仅在自愈脚本中注入

指纹记录与恢复代码仅注入到包含多策略交互步骤的生成脚本中。纯导航脚本不注入。

#### 场景:有自愈步骤时注入指纹常量

- **当** `generate_script()` 检测到 `has_healing=True`（存在 >1 条策略的交互步骤）
- **那么** 生成脚本包含 `FINGERPRINT_FILE` 路径常量和指纹操作的 helper 函数
- **那么** `_safe()` 函数签名和逻辑包含指纹记录与恢复分支

#### 场景:无自愈步骤时不注入

- **当** `has_healing=False`
- **那么** 生成脚本不包含任何指纹相关代码
